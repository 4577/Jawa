Various utility classes and methods to make life a little bit easier.

ezip Module¶

class jawa.util.ezip.EditableZipFile(io=None)[source]¶

Bases: object

A light wrapper around the python zipfile.ZipFile that enables in-memory modification.

If io is provided, its contents are loaded. io may be a file-system path or a file-like object.

close(*exc_info)¶

Closes the JarFile and any underlying objects.

namelist[source]¶

Returns a list of all paths in this JarFile.

read(path)[source]¶

Returns the contents of path, if it exists, otherwise None.

regex(regex)[source]¶

Returns an iterator over every path that matches regex. For example, to get all files ending in .class that aren’t in a sub-directory:

>>> for path in jf.regex('[^/]+\.class'):
...    print(path)
remove(path)[source]¶

Removes a file from the JarFile.

>>> print jf.namelist()
['my_file.txt', 'that_other_file.txt', ...]
>>> jf.remove('my_file.txt')
>>> print jf.namelist()
['that_other_file.txt', ...]

jf Module¶

Tools for working with JVM .jar archives, which are simply ZIP archives with some mandatory structure.

Note

If czipfile is available, decompression will be noticably faster.

class jawa.util.jf.JarFile(io=None)[source]¶

Bases: jawa.util.ezip.EditableZipFile

Provides Jawa-specific helpers around EditableZipFile.

As of 0.1.0, jawa.util.jf.JarFile is also a context manager.

>>> from jawa.util.jf import JarFile
>>> with JarFile(sys.argv[1]) as jf:
...    print(jf.namelist())
